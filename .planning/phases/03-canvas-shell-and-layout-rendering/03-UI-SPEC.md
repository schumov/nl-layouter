## Spacing Scale

Declared values вҖ” multiples of 4 only:

| Token | Value | Tailwind Class | Phase 3 Usage |
|-------|-------|----------------|---------------|
| xs | 4px | `p-1` / `gap-1` | вҖ” (not used in Phase 3) |
| sm | 8px | `p-2` / `gap-2` / `space-y-2` | ColumnGrid inner padding + gap; RowBlock list spacing |

| lg | 24px | `p-6` | вҖ” (palette tab content: `p-4` = 16px, see note) |

> **Palette tab content padding:** `p-4` (16px) вҖ” matches `md` token, consistent with

### RowBlock вҖ” Section Wrapper

```
div.bg-white.rounded.border.shadow-sm.overflow-hidden
  в””в”Җв”Җ ColumnGrid
```

- Background: `bg-white` (explicit, overrides any parent bg-canvas)
- Border: `border` (uses `--border` token вҶ’ `#e5e5e5`)
- Shadow: `shadow-sm`
- Radius: `rounded` (uses `--radius` вҶ’ `0.625rem` / 10px)
- `overflow-hidden` вҖ” clips column content at card edges

> Section-level `backgroundColor` and `paddingTop`/`paddingBottom` are applied via
> inline `style` prop to support future section styling configuration.
> Default: backgroundColor `#ffffff`, no padding overrides.

### ColumnGrid вҖ” Layout Type вҶ’ Flex Proportions (LAYOUT-01 through LAYOUT-05)

```
div.flex.gap-2.p-2
  в””в”Җв”Җ div[data-testid="column-wrapper"].min-w-0.{basis-class} Г— N
        в””в”Җв”Җ ColumnSlot
```

**Static `COLUMN_CLASSES` record вҖ” every class name is a complete string literal:**

| LayoutType | Column 1 | Column 2 | Column 3 | Visual |
|------------|----------|----------|----------|--------|
| `1col` | `basis-full` | вҖ” | вҖ” | `[100%]` |
| `2col` | `basis-1/2` | `basis-1/2` | вҖ” | `[50%][50%]` |
| `3col` | `basis-1/3` | `basis-1/3` | `basis-1/3` | `[33%][33%][33%]` |
| `small-left-big-right` | `basis-1/3` | `basis-2/3` | вҖ” | `[33%][67%]` |
| `big-left-small-right` | `basis-2/3` | `basis-1/3` | вҖ” | `[67%][33%]` |

> вҡ пёҸ **CRITICAL вҖ” Tailwind v4 JIT rule:** Never construct class names via template literals
> (e.g. `` `basis-${fraction}` ``). The JIT scanner requires complete string literals.
> All 8 unique `basis-*` classes above must appear verbatim in `COLUMN_CLASSES`.
> [Source: RESEARCH.md Pitfall 1]

> `min-w-0` is mandatory on every column wrapper div вҖ” prevents flex overflow when slot
> content is wider than the column's flex-basis.

### BuilderPalette вҖ” Tabs Shell

```
div.flex-[2].min-w-0.border-l.bg-background.overflow-y-auto
  в””в”Җв”Җ Tabs[defaultValue="layouts"].h-full.flex.flex-col
        в””в”Җв”Җ TabsList.w-full.shrink-0.rounded-none.border-b
              в”ңв”Җв”Җ TabsTrigger[value="layouts"].flex-1  вҶ’  "Layouts"
              в””в”Җв”Җ TabsTrigger[value="elements"].flex-1  вҶ’  "Elements"
        в”ңв”Җв”Җ TabsContent[value="layouts"].p-4.space-y-2
        в”Ӯ     в””в”Җв”Җ LayoutCard Г— 5
        в””в”Җв”Җ TabsContent[value="elements"].p-4
              в””в”Җв”Җ stub paragraph
```

> `rounded-none` on `TabsList` вҖ” overrides shadcn default radius so the tab bar sits
> flush against the palette's top edge (no gap between border-l panel edge and tab bar).
> `border-b` on `TabsList` вҖ” visual separator between tabs and content.
> `shrink-0` on `TabsList` вҖ” prevents the tab bar from shrinking when palette content is tall.

## Interaction Contract

| Interaction | Behavior | Phase |
|-------------|----------|-------|
| Palette tab switch | Click "Layouts" вҶ” "Elements" вҖ” standard shadcn Tabs keyboard + click | Phase 3 |
| Layout cards | Non-interactive (display only, `cursor-default`) | Phase 3 вҶ’ Phase 4 wires DnD |

- `role="tablist"` on `TabsList`
- `role="tab"` + `aria-selected` on each `TabsTrigger`
- `role="tabpanel"` on each `TabsContent`
- Full keyboard navigation (Arrow keys, Home, End) вҖ” provided by Radix UI primitive
