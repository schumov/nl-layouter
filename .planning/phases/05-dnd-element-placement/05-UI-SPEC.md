## Spacing Scale

Carries forward Phase 4 scale. Phase 5 usages added:

| Token | px | Tailwind | Phase 5 Usage |
|-------|----|----------|---------------|
| xs | 4px | `gap-1`, `p-1`, `top-1`, `right-1` | × button position inside slot (`top-1 right-1`); remove-confirm button gap (`gap-1`) |
| sm | 8px | `gap-2`, `p-2`, `space-y-2` | Element card icon-label gap (`gap-2`); Elements tab card list spacing (`space-y-2`) |
| md | 16px | `p-4`, `px-4` | Elements tab content padding (`p-4`); inspector body padding (`p-4`); inspector header horizontal padding (`px-4`) |
| lg | 24px | `py-6` | Standard lg token (24px) — not directly used in Phase 5 components (see inspector exception below) |
| xl | 32px | `py-8` | Canvas vertical padding (Phase 3 — unchanged) |
| 2xl | 48px | `h-48` | Empty canvas drop zone height (Phase 4 — unchanged) |
| 3xl | 64px | `py-16` | Empty canvas loading state padding (Phase 3 — unchanged) |

**Exceptions:**

| Element | Value | Rationale |
|---------|-------|-----------|
| ColumnSlot minimum height | 80px — `min-h-[80px]` | Retained from Phase 3; Phase 5 inherits unchanged |
| × button touch target | 20px — `size-5` | Slot interior; 32px (`icon-sm`) would consume ≥16% of a narrow 3-column slot width at 640px max-width. Desktop-only tool; 20px is acceptable. Not a spacing token — an explicit exception. |
| Inspector header padding | `py-3` (12px) | 12px is not in the 4px grid but matches shadcn panel header conventions and `border-b` visual rhythm; `py-2` (8px) is too tight, `py-4` (16px) too tall. Declared exception. |

## Typography

No new type roles introduced beyond Phase 4. Phase 5 reuses the existing 3-size scale:

| Role | Size | Tailwind | Weight | Line Height | Phase 5 Usage |
|------|------|----------|--------|-------------|---------------|
| Body | 14px | `text-sm` | 400 regular | 1.5 | Element card labels; inspector note text; empty slot hint text (unchanged) |
| Label/Title | 12–16px | `text-xs` / `text-sm` / `text-base` | 600 semibold | 1.2–1.4 | Remove confirm "Remove?" text; inspector panel header; BuilderHeader |

**Two weights only:** `400 regular` for body/muted text; `600 semibold` for all labels, headings, and actionable text. Weight 500 (medium) is not used in Phase 5.

**Inspector panel header text:** `text-sm font-semibold` (14px / weight 600) — consistent with label role above; not a new type role.

**No heading or display sizes introduced in Phase 5.** Element content typography belongs to Phases 6–7.

## Color

Carries forward all Phase 4 tokens plus one new semantic entry for element drag-over:

| Role | Value | Tailwind Token | Usage |
|------|-------|----------------|-------|
| Dominant (60%) | `#f4f4f5` | `bg-canvas` (`--color-canvas`) | Canvas panel background (unchanged from Phase 3) |
| Secondary (30%) | `#ffffff` | `bg-white` / `bg-background` | RowBlock surface, palette surface, inspector surface |
| Border / muted | `oklch(0.922 0 0)` ≈ `#e4e4e7` | `border` (`--color-border`) | Slot borders, card borders, inspector header `border-b` |
| Layout drag-over (blue) | `#eff6ff` fill / `#60a5fa` border | `bg-blue-50 border-blue-400` | Empty canvas drop zone `isOver` ONLY (Phase 4 — unchanged) |
| **Element drag-over (green)** | `#f0fdf4` fill / `#4ade80` border | `bg-green-50 border-green-400` | **Empty ColumnSlot `isOver` state ONLY — D-03** |
| Selection ring | `oklch(0.484 0.209 257.0)` ≈ `#0066cc` | `ring-ring` (`--ring`) | Selected occupied slot ring ONLY |
| Destructive | `oklch(0.577 0.245 27.325)` ≈ `#dc2626` | `text-destructive` | Remove confirm "Remove?" text; × hover state |
| Muted foreground | `oklch(0.556 0 0)` ≈ `#71717a` | `text-muted-foreground` | Element card labels (idle); inspector note text; × idle state; Remove confirm "Cancel" |
| Accent fill | `oklch(0.97 0 0)` ≈ `#f5f5f5` | `bg-accent` | × button hover background fill |

**Accent reserved for:**
- Hover background on all interactive surface controls: element palette cards (`hover:bg-accent hover:text-accent-foreground`), × remove button (`hover:bg-accent`), and ghost icon buttons such as the back arrow (`hover:bg-accent hover:text-accent-foreground` via shadcn `variant="ghost"`)

**Green reserved for:**
- Empty ColumnSlot `isOver` state ONLY — never used for any other interactive or visual state in Phase 5

**Blue (layout drag-over) reserved for:**
- Empty canvas drop zone `isOver` state ONLY — unchanged from Phase 4

**Selection ring (brand blue) reserved for:**
- Selected occupied slot `ring-2 ring-ring ring-inset` — one state only

**Design rationale — colour-coded drag types:**
- Blue (`bg-blue-50 / border-blue-400`) = layout drop targets (canvas-level)
- Green (`bg-green-50 / border-green-400`) = element drop targets (slot-level)
- The colour separation makes the two DnD systems visually distinguishable at a glance.

### 2. `ColumnSlot` (upgraded from Phase 3)
**File:** `apps/client/src/components/builder/ColumnSlot.tsx`

**New props:**
```typescript
interface ColumnSlotProps {
  slot:        ColumnSlotData;
  sectionId:   string;           // required in Phase 5 — no longer a stub
  isSelected?: boolean;          // slot.id === selectedElementId from Zustand
  onSelect?:   () => void;       // → setSelectedElement(slot.id); called on occupied slot click
}
```

**useDroppable registration:**
```typescript
const { setNodeRef, isOver } = useDroppable({
  id:   slot.id,
  data: { type: DRAG_TYPES.ELEMENT_CARD },  // marks this node as COLUMN_SLOT target
});
```

**Local state:**
```typescript
const [isConfirming, setIsConfirming] = useState(false);  // per-slot remove confirm
```

**State machine — complete enumeration:**

#### State 1: Empty — Idle
Condition: `!slot.element && !isOver`
```
div[ref=setNodeRef]
  className="min-h-[80px] flex items-center justify-center
             border-2 border-dashed border-border
             rounded text-sm text-muted-foreground select-none"
```
Content: `"Drop element here"`

#### State 2: Empty — Drag-Over (ELEMENT_CARD hovering)
Condition: `!slot.element && isOver`
```
div[ref=setNodeRef]
  className="min-h-[80px] flex items-center justify-center
             border-2 border-green-400 bg-green-50
             rounded text-sm text-muted-foreground select-none
             transition-colors duration-150"
```
Content: `"Drop element here"` — **unchanged** (D-03)
Note: `border-dashed` dropped; `border-2` retained. Background fill animates in via `transition-colors`.

#### State 3: Occupied — Idle (no hover, not selected)
Condition: `slot.element && !isSelected`
```
div[ref=setNodeRef]
  className="relative group min-h-[80px] rounded overflow-hidden cursor-pointer"
  onClick={(e) => { e.stopPropagation(); onSelect?.(); }}
```
Children:
- `<ElementRenderer element={slot.element} />` (unchanged stub from Phase 3)
- × button: `opacity-0 transition-opacity duration-100`

#### State 4: Occupied — Hover (cursor over slot, not selected)
Condition: `slot.element && !isSelected && group-hover`
- × button becomes visible via CSS: `opacity-0 group-hover:opacity-100`
- No other visual change to slot wrapper

#### State 5: Occupied — Selected
Condition: `slot.element && isSelected`
```
div[ref=setNodeRef]
  className="relative group min-h-[80px] rounded overflow-hidden cursor-pointer
             ring-2 ring-ring ring-inset"
```
- × button: `opacity-100` (explicit via `cn(...)` — overrides `opacity-0` base)

#### State 6: Remove Confirm
Condition: `slot.element && isConfirming`
- Slot wrapper unchanged from State 3/4/5
- × button replaced by inline confirm (see × button spec below)

**× Button spec (when `!isConfirming`):**
```
button
  className="absolute top-1 right-1
             flex items-center justify-center size-5 rounded
             text-muted-foreground hover:text-destructive hover:bg-accent
             transition-colors duration-100
             opacity-0 group-hover:opacity-100
             [isSelected && 'opacity-100']"
  aria-label="Remove element"
  onClick={(e) => { e.stopPropagation(); setIsConfirming(true); }}
```
Icon: `<X className="size-3" />`

Full classes via `cn`:
```typescript
cn(
  'absolute top-1 right-1 flex items-center justify-center size-5 rounded',
  'text-muted-foreground hover:text-destructive hover:bg-accent',
  'transition-colors duration-100',
  'opacity-0 group-hover:opacity-100 transition-opacity duration-100',
  isSelected && 'opacity-100',
)
```

**Remove confirm spec (when `isConfirming`):**
Replaces × button at same `absolute top-1 right-1` position:
```
div
  aria-live="polite"
  className="absolute top-1 right-1 flex items-center gap-1"
```
Children:
- `"Remove?"` button: `text-xs font-semibold text-destructive hover:underline cursor-pointer whitespace-nowrap`
  - `onClick={(e) => { e.stopPropagation(); removeElement(slot.id); }}` — dispatches store action
- `"Cancel"` button: `text-xs text-muted-foreground hover:text-foreground cursor-pointer whitespace-nowrap`
  - `onClick={(e) => { e.stopPropagation(); setIsConfirming(false); }}`

**Occupied slots during ELEMENT_CARD drag:**
- No visual change (D-02). `isOver` is `true` only when the custom collision detection returns this slot as the target, which is based on cursor proximity — occupied slots never receive the green highlight, only empty ones conditionally highlight based on `isOver && !slot.element`.

**Canvas click-outside deselect (D-06):**
- `ColumnSlot` calls `e.stopPropagation()` on all occupied slot clicks to prevent bubbling
- `BuilderCanvas` wrapper div receives `onClick={() => setSelectedElement(null)}`
- Any click on canvas background (not an occupied slot) bubbles to BuilderCanvas and clears selection

### 4. `BuilderPage` — Right Panel Conditional Rendering
**File:** `apps/client/src/pages/BuilderPage.tsx`

**New Zustand selectors needed:**
```typescript
const selectedElementId   = useNewsletterStore(s => s.selectedElementId);
const setSelectedElement  = useNewsletterStore(s => s.setSelectedElement);

// Derive element type for InspectorPanel from selectedElementId:
const selectedElementType = useNewsletterStore(s => {
  if (!s.selectedElementId || !s.doc) return null;
  for (const row of s.doc.rows) {
    for (const slot of row.slots) {
      if (slot.id === s.selectedElementId && slot.element) {
        return slot.element.type;
      }
    }
  }
  return null;
});
```

**Updated `<main>` render (conditional right panel — D-04, D-05, D-06):**
```
<main className="flex flex-1 overflow-hidden">
  <BuilderCanvas
    doc={doc}
    onCanvasClick={() => setSelectedElement(null)}
  />
  {selectedElementId && selectedElementType
    ? <InspectorPanel
        elementType={selectedElementType}
        onBack={() => setSelectedElement(null)}
      />
    : <BuilderPalette />
  }
</main>
```

**Visual contract:**
- Right panel occupies `flex-[2] min-w-0` — same dimensions for both `BuilderPalette` and `InspectorPanel`
- Panel swap is instantaneous (no animation/transition in Phase 5)
- `BuilderCanvas` requires `onCanvasClick` prop added in Phase 5 (new optional prop)

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Element card keyboard drag | `{...attributes}` from `useDraggable` — Radix/dnd-kit adds `role="button"` + `tabIndex` automatically |
| × button label | `aria-label="Remove element"` |
| Remove confirm live region | `aria-live="polite"` on the confirm div — announces "Remove? Cancel" when shown |
| Focus management — remove confirm | "Cancel" button receives focus when confirm state appears (via `autoFocus` or `useEffect` focus) |
| Inspector back arrow label | `aria-label="Back to palette"` on Button |
| Occupied slot click | Slot div adds `tabIndex={0}` for keyboard access; `onKeyDown` handles `Enter`/`Space` → `onSelect?.()` |
| Canvas click-outside | `onClick` on `BuilderCanvas` wrapper — does not need an aria label (decorative container) |

## Visual Focal Point

**Primary visual anchor (idle state):** The canvas section columns and their slots. As elements are placed, slot content progressively fills the canvas — the empty dashed borders recede, replaced by `ElementRenderer` content.

**During element drag:** The DragOverlay ghost (element card with icon + label) becomes the primary focal point temporarily. Source card fades to `opacity-40`. Target empty slots show green `bg-green-50 / border-green-400` on hover.

**Selected state:** The brand blue `ring-2 ring-ring ring-inset` on the occupied slot signals selection without affecting layout. The right panel swaps from palette to inspector, reinforcing the selection context.
