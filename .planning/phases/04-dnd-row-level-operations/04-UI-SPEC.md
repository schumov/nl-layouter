# Phase 4 — UI Design Contract: DnD Row-Level Operations

> Visual and interaction contract for Phase 4 DnD. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Requirements covered:** CANVAS-02, CANVAS-03, CANVAS-04, CANVAS-05, CANVAS-06

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui |
| Style | new-york |
| Preset | neutral base, CSS variables, no prefix |
| Component library | Radix UI (via shadcn) |
| Icon library | lucide-react (`GripVertical`, `Copy`, `Trash2`) |
| Font | System default (Tailwind/shadcn default sans-serif stack) |
| CSS engine | Tailwind CSS v4 — CSS-first, `@theme inline` tokens, no `tailwind.config.*` file |

**Tailwind v4 rule (locked — CC-5 equivalent):** All class names MUST be complete string literals. Never build via template literals. Source: `ColumnGrid.tsx` existing pattern.

## Spacing Scale

Declared values (all multiples of 4):

| Token | px | Tailwind | Usage in Phase 4 |
|-------|----|----------|-----------------|
| xs | 4px | `gap-1`, `p-1` | Section controls inner gap between icon buttons |
| sm | 8px | `gap-2`, `p-2`, `space-y-2` | Canvas row gap (existing `space-y-2`), ColumnGrid inner padding |
| md | 16px | `px-4`, `gap-4` | Canvas horizontal padding (existing `px-4`) |
| lg | 24px | `p-6` | Not used in Phase 4 |
| xl | 32px | `py-8` | Canvas vertical padding (existing `py-8`) |
| 2xl | 48px | `h-48` | Empty canvas drop zone height (`h-48` = 192px, multiple of 4) |
| 3xl | 64px | `py-16` | Existing empty state padding (Phase 3 pattern retained for loading state) |

**Exceptions:**
- Section controls cluster: `pl-2` (8px) left padding on the floating cluster to create visual breathing room from the RowBlock edge.
- Drag activation constraint: 8px — not a visual spacing token, but a UX threshold.
- Touch target minimum for icon buttons: 32px (`icon-sm` = `size-8`) — acceptable for a desktop builder tool. Accessibility: keyboard navigation covers non-pointer users.

## Typography

All text in Phase 4 UI elements uses the existing `text-sm` / `text-xs` scale. No new type roles introduced.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 | Palette card labels, empty drop zone hint text |
| Label | 12px (`text-xs`) | 500 (medium) | 1.4 | Inline delete-confirm text ("Delete?" / "Cancel") |
| Muted | 14px (`text-sm`) | 400 (regular) | 1.5 | "Drop a layout here" centered drop zone text |

**No heading or display sizes introduced in Phase 4.** Section content typography belongs to Phases 6–7.

## Color

| Role | Value | Tailwind Token | Usage |
|------|-------|----------------|-------|
| Dominant (60%) | `#f4f4f5` | `bg-canvas` (`--color-canvas`) | Canvas panel background (existing) |
| Secondary (30%) | `#ffffff` | `bg-white` / `bg-background` | RowBlock surface, palette surface, ghost overlay bg |
| Border / muted | `oklch(0.922 0 0)` ≈ `#e4e4e7` | `border` (`--color-border`) | RowBlock border, palette card border, controls hover |
| Accent (10%) — drag-over | `#eff6ff` fill / `#60a5fa` border | `bg-blue-50` / `border-blue-400` | Drop zone active state ONLY |
| Drop zone default border | `#d4d4d4` | `border-neutral-300` | Empty canvas drop zone default dashed border |
| Destructive | `oklch(0.577 0.245 27.325)` ≈ `#dc2626` | `text-destructive` / `bg-destructive` | Delete confirm "Delete?" label, Trash2 hover |
| Muted foreground | `oklch(0.556 0 0)` ≈ `#71717a` | `text-muted-foreground` | Controls icons default tint, drop zone hint text |

**Accent reserved for:**
- Empty canvas drop zone `isOver` state only — `bg-blue-50 border-blue-400`
- NO other element in Phase 4 uses blue-50 / blue-400

**Note on drag-over blue vs brand primary:** The drag-over state uses Tailwind's standard `blue-50` / `blue-400` (lighter, softer). The brand Infineon Blue (`#0066cc`, `--primary`) is NOT used for drag-over — it would be too saturated for a large fill area. Consistent with CONTEXT.md D-04.

**Destructive reserved for:**
- Delete button Trash2 icon `hover:text-destructive`
- Inline delete-confirm "Delete?" text (`text-destructive`)

## Copywriting Contract

| Element | Copy | Notes |
|---------|------|-------|
| Empty canvas drop zone | "Drop a layout here" | Centered in `h-48` zone; `text-sm text-muted-foreground`; replaces Phase 3 text-only state |
| Empty canvas — after drop zone (loading state) | "No sections yet. Drag a layout from the palette to begin." | Retained from Phase 3 for the doc-loading/null state only |
| Delete button aria-label | `"Delete section"` | Screen reader label on the Trash2 icon button |
| Duplicate button aria-label | `"Duplicate section"` | Screen reader label on the Copy icon button |
| Drag handle aria-label | `"Drag to reorder"` | Screen reader label on the GripVertical icon button |
| Inline delete confirm — primary | `"Delete?"` | `text-destructive font-medium text-xs`; click dispatches `removeSection` |
| Inline delete confirm — cancel | `"Cancel"` | `text-muted-foreground text-xs`; click resets local confirm state |
| Drag ghost — LAYOUT_CARD | Layout name (e.g. `"1 Column"`, `"2 Columns"`) | Same label text as palette card; visible in ghost overlay |
| Palette card cursor affordance | `cursor-grab` | CSS only — no copy change |

**No destructive modal introduced.** Delete confirmation is inline within the section controls cluster (D-07, local `useState` per RowBlock). No `AlertDialog` or modal.

## Component Inventory

### New Components

#### 1. `DragDropProvider`
**File:** `apps/client/src/components/builder/DragDropProvider.tsx`

Purpose: Wraps `BuilderPage` in `DndContext`. Owns all drag event handlers (`onDragStart`, `onDragEnd`). Renders the `DragOverlay` for both ghost types.

Visual contract:
- `DragOverlay` is rendered at the root of this provider
- Ghost for `LAYOUT_CARD`: `<div className="p-3 border rounded-md text-sm bg-white shadow-md opacity-80 cursor-grabbing select-none">` containing the layout name label
- Ghost for `CANVAS_ROW`: `<div className="opacity-80 cursor-grabbing"><RowBlock section={activeSection} /></div>` — identical visual to a real row but lifted and semi-transparent

#### 2. `SortableRowList`
**File:** `apps/client/src/components/builder/SortableRowList.tsx`

Purpose: Wraps section array in `SortableContext` with `verticalListSortingStrategy`. Renders `SortableRowBlock` (RowBlock with useSortable). Conditionally renders the empty canvas drop zone when `rows.length === 0`.

Visual contract:

**Empty drop zone — default state:**
```
┌ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┐  h-48
│                                                       │  border-2 border-dashed border-neutral-300 rounded-md
│              Drop a layout here                       │  text-sm text-muted-foreground text-center
│                                                       │  flex items-center justify-center
└ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┘
```

**Empty drop zone — drag-over state (isOver = true):**
```
┌───────────────────────────────────────────────────────┐  h-48
│                                                       │  border-2 border-blue-400 rounded-md bg-blue-50
│              Drop a layout here                       │  text-sm text-muted-foreground text-center
│                                                       │  transition-colors duration-150
└───────────────────────────────────────────────────────┘
```

Class swap: `border-neutral-300 border-dashed` → `border-blue-400` (solid) + add `bg-blue-50`
Transition: `transition-colors duration-150` (subtle, not jarring)

#### 3. Updated `RowBlock` (with `SectionControls`)
**File:** `apps/client/src/components/builder/RowBlock.tsx`

Phase 4 extends `RowBlock` to:
1. Accept `useSortable` bindings as props
2. Apply CSS transform/transition from sortable
3. Render `SectionControls` floating outside the right edge

**RowBlock wrapper layout:**
```
┌──────────────────────────────────────────────────────────┐ ←── relative positioning
│  bg-white rounded border shadow-sm overflow-hidden        │ ←── existing RowBlock styles
│  ┌──────────────────────────────────────────────────────┐ │
│  │ ColumnGrid content (unchanged)                       │ │
│  └──────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────┘
                                                   ┌──────────────┐ ←── absolute right-0 top-1/2
                                                   │ SectionControls │    -translate-y-1/2 translate-x-full
                                                   └──────────────┘     pl-2 flex items-center gap-1
```

**RowBlock dragging state (`isDragging = true`):**
- Apply `opacity-40` to the source RowBlock while the drag overlay is visible
- `transition-opacity duration-150`

**Props added to RowBlock:**
```typescript
interface RowBlockProps {
  section:      Section;
  // sortable bindings (all optional — default/no-op when not under SortableContext)
  listeners?:   SyntheticListenerMap | undefined;
  attributes?:  DraggableAttributes;
  setNodeRef?:  (node: HTMLElement | null) => void;
  style?:       React.CSSProperties;  // transform + transition from useSortable
  isDragging?:  boolean;
  onDuplicate?: () => void;
  onDelete?:    () => void;
}
```

#### 4. `SectionControls`
**File:** Inlined in `RowBlock.tsx` as a sub-component `SectionControls` (not a separate file — scoped to RowBlock)

**Normal state — three buttons:**
```
[≡ GripVertical]  [⧉ Copy]  [🗑 Trash2]
   cursor-grab       ghost      ghost
   icon-sm           icon-sm    icon-sm
   muted-fg          muted-fg   muted-fg / hover:text-destructive
```

Tailwind classes for the cluster wrapper:
```
absolute right-0 top-1/2 -translate-y-1/2 translate-x-full pl-2
flex items-center gap-1
```

Each button: `<Button variant="ghost" size="icon-sm" ...>`
- **GripVertical button:** `cursor-grab` overriding button default; `text-muted-foreground`; spreads `...listeners` from useSortable so it becomes the actual drag handle
- **Copy button:** `text-muted-foreground hover:text-foreground`; on click → `onDuplicate()`
- **Trash2 button:** `text-muted-foreground hover:text-destructive`; on click → set local `isConfirming = true`

**Delete-confirm state — replaces Trash2 button:**
```
[≡ GripVertical]  [⧉ Copy]  [Delete?]  [Cancel]
                              text-xs     text-xs
                              text-destr  text-muted
                              font-medium font-normal
                              cursor-ptr  cursor-ptr
```

The "Delete?" and "Cancel" are plain `<button>` or `<span role="button">` elements (not full shadcn Button since they're text-only, inline, xs size):
- `"Delete?"`: `text-xs font-medium text-destructive hover:underline cursor-pointer whitespace-nowrap`
- `"Cancel"`: `text-xs text-muted-foreground hover:text-foreground cursor-pointer whitespace-nowrap`

State transitions:
- `isConfirming = false` → normal state (GripVertical + Copy + Trash2)
- `isConfirming = true` → confirm state (GripVertical + Copy + "Delete?" + "Cancel")
- Click "Delete?" → dispatch `removeSection(section.id)` → component unmounts (state irrelevant)
- Click "Cancel" → `setIsConfirming(false)` → back to normal state
- State: local `const [isConfirming, setIsConfirming] = useState(false)` in RowBlock component body

## Interaction State Matrix

Complete enumeration of all interactive states in Phase 4.

### Palette Layout Cards

| State | Visual |
|-------|--------|
| Default (idle) | `p-3 border rounded-md text-sm cursor-grab bg-background` |
| Hover | `hover:bg-accent hover:text-accent-foreground` — subtle fill |
| Active drag source | `opacity-40 cursor-grabbing` — source fades behind overlay |
| Ghost overlay | `p-3 border rounded-md text-sm bg-white shadow-md opacity-80 cursor-grabbing` |

### Canvas Drop Zone (empty canvas only)

| State | Visual |
|-------|--------|
| Default (no drag in progress) | `h-48 border-2 border-dashed border-neutral-300 rounded-md flex items-center justify-center text-sm text-muted-foreground` |
| Drag over (isOver = true) | `h-48 border-2 border-blue-400 rounded-md flex items-center justify-center text-sm text-muted-foreground bg-blue-50 transition-colors duration-150` |

### Canvas Row (RowBlock with useSortable)

| State | Visual |
|-------|--------|
| Default (idle) | `bg-white rounded border shadow-sm overflow-hidden` (existing) |
| Drag source (being sorted) | `opacity-40` added — the DragOverlay clone is the "real" visual |
| Sortable placeholder | dnd-kit CSS transform handles visual reflow automatically |
| Drag overlay clone | `opacity-80 cursor-grabbing` wrapper around full RowBlock |

### Section Controls — GripVertical Handle

| State | Visual |
|-------|--------|
| Default | `Button variant="ghost" size="icon-sm"` — `text-muted-foreground` |
| Hover | shadcn ghost hover: `hover:bg-accent hover:text-accent-foreground` |
| Active drag | `cursor-grabbing` |

### Section Controls — Duplicate Button

| State | Visual |
|-------|--------|
| Default | `Button variant="ghost" size="icon-sm" text-muted-foreground` |
| Hover | `hover:text-foreground` (inherits ghost hover bg) |
| Click feedback | Button active state (shadcn handles) |

### Section Controls — Delete Button (normal → confirm flow)

| State | Visual |
|-------|--------|
| Default | `Button variant="ghost" size="icon-sm" text-muted-foreground` |
| Hover | `hover:text-destructive` |
| Click (1st) | Transitions to confirm state — replaces Trash2 button with text pair |
| Confirm — "Delete?" | `text-xs font-medium text-destructive hover:underline cursor-pointer` |
| Confirm — "Cancel" | `text-xs text-muted-foreground hover:text-foreground cursor-pointer` |
| Click "Delete?" | Dispatches `removeSection` — RowBlock unmounts |
| Click "Cancel" | Returns to default state |

## Visual Focal Point

**Primary visual anchor (idle state):** The RowBlock stack is the main content object — the repeating white card rows dominate the canvas surface. `SectionControls` is the secondary visual layer, rendered outside the block to the right and intentionally lighter (`text-muted-foreground`) to not compete with content.

**Drag-active state:** The `DragOverlay` ghost becomes the primary focal point temporarily; source rows and palette card fade to `opacity-40` to reinforce this hierarchy shift.
