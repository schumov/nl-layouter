# Plan 09-04: DOCTYPE + MSO Head Wrapper

## Goal

Create the `doctype.ts` utility that transforms the HTML string produced by react-email's
`render()` + juice into a proper email-client-compatible document. Specifically: replaces
HTML5 DOCTYPE with XHTML 1.0 Transitional + VML namespaces, and injects the MSO Outlook
reset block into `<head>`. This is a standalone, dependency-free string transformer.

## Context

react-email's `render()` produces `<!DOCTYPE html>` (HTML5) with `<html lang="en">`. Email
clients — especially Outlook — require XHTML Transitional DOCTYPE and VML namespace attributes
on `<html>` for conditional comments and VML shapes to work correctly. This transformation
happens **after** juice inlines the CSS (Plan 09-05 calls this function).

This plan is standalone and has no dependency on pipeline.ts — `wrapWithDoctype` is a pure
function that Plan 09-05 imports.

## Tasks

### Task 1: Create `doctype.ts`

Create `apps/server/src/export/doctype.ts`:

```typescript
// apps/server/src/export/doctype.ts
// Pure string transformation: converts react-email HTML output to email-client-ready HTML.
//
// Transformations applied (in order):
//   1. Replace HTML5 DOCTYPE with XHTML 1.0 Transitional
//   2. Replace <html ...> opening tag with VML-namespace variant
//   3. Set <title> in <head> (if title param provided)
//   4. Inject MSO OfficeDocumentSettings block after <head> open tag
//
// This function is called by pipeline.ts as the LAST step of the pipeline,
// AFTER react-email render() and AFTER juice CSS inlining.
//
// IMPORTANT: This function does NOT process multi-column MSO wrappers.
// Those are injected by documentToEmailTree.tsx (Plan 09-03) during component rendering.

// ── DOCTYPE and namespace constants ─────────────────────────────────────────

const XHTML_DOCTYPE = `<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN"
  "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">`;

// VML namespaces required for Outlook conditional comments and VML shapes.
// xmlns:v = VML namespace
// xmlns:o = Office namespace (used by o:OfficeDocumentSettings etc.)
const HTML_OPEN_WITH_VML =
  '<html xmlns="http://www.w3.org/1999/xhtml" ' +
  'xmlns:v="urn:schemas-microsoft-com:vml" ' +
  'xmlns:o="urn:schemas-microsoft-com:office:office" ' +
  'lang="en">';

// MSO reset block — injected immediately after <head> opening tag.
// o:AllowPNG         — enables PNG transparency support in Outlook
// o:PixelsPerInch 96 — enforces 96 DPI rendering (prevents Outlook scaling images at 72 DPI)
const MSO_RESET_BLOCK =
  '<!--[if mso]>' +
  '<noscript>' +
  '<xml>' +
  '<o:OfficeDocumentSettings>' +
  '<o:AllowPNG/>' +
  '<o:PixelsPerInch>96</o:PixelsPerInch>' +
  '</o:OfficeDocumentSettings>' +
  '</xml>' +
  '</noscript>' +
  '<![endif]-->';

// ── Helper: find and replace the first occurrence of a pattern ───────────────

/**
 * Replace the first regex match in a string. Throws if the pattern is not found
 * and `required` is true; silently skips if `required` is false.
 */
function replaceFirst(
  html: string,
  pattern: RegExp,
  replacement: string,
  required = true,
): string {
  if (!pattern.test(html)) {
    if (required) {
      console.warn(`[wrapWithDoctype] Expected pattern not found: ${pattern.toString()}`);
    }
    return html;
  }
  return html.replace(pattern, replacement);
}

// ── Public API ───────────────────────────────────────────────────────────────

/**
 * Transform a react-email HTML string into an email-client-ready document.
 *
 * @param html   The HTML string produced by react-email render() + juice.
 * @param title  Optional document title to set in <title> element.
 * @returns      Modified HTML string with XHTML DOCTYPE, VML namespaces, and MSO head block.
 */
export function wrapWithDoctype(html: string, title?: string): string {
  let result = html;

  // ── Step 1: Replace DOCTYPE ───────────────────────────────────────────────
  // react-email produces: <!DOCTYPE html>
  // Also handle: <!DOCTYPE html PUBLIC ...> (react-email may vary across versions)
  result = replaceFirst(
    result,
    /<!DOCTYPE[^>]*>/i,
    XHTML_DOCTYPE,
  );

  // ── Step 2: Replace <html> opening tag ───────────────────────────────────
  // react-email produces: <html lang="en"> or <html lang="en" dir="ltr"> etc.
  // We replace the entire opening <html ...> tag (not self-closing, not </html>).
  result = replaceFirst(
    result,
    /<html[^>]*>/i,
    HTML_OPEN_WITH_VML,
  );

  // ── Step 3: Set <title> (if provided) ────────────────────────────────────
  // react-email may produce <title></title>. Replace it with the newsletter title.
  if (title) {
    const escapedTitle = title
      .replace(/&/g, '&amp;')
      .replace(/</g, '&lt;')
      .replace(/>/g, '&gt;')
      .replace(/"/g, '&quot;');

    // Replace existing <title>...</title> content
    if (/<title>/i.test(result)) {
      result = result.replace(/<title>[^<]*<\/title>/i, `<title>${escapedTitle}</title>`);
    } else {
      // If no <title> element, inject after <head>
      result = result.replace(/<head>/i, `<head><title>${escapedTitle}</title>`);
    }
  }

  // ── Step 4: Inject MSO reset block after <head> ───────────────────────────
  // Placement: immediately after <head> opening tag.
  // This must come AFTER the <title> injection in step 3.
  result = replaceFirst(
    result,
    /<head>/i,
    `<head>${MSO_RESET_BLOCK}`,
  );

  return result;
}
```

**Implementation notes:**

1. **Pattern matching robustness:** `react-email@^2.x` may produce slightly different DOCTYPE
   or `<html>` attribute orderings than `@^1.x`. Using `/<!DOCTYPE[^>]*>/i` and `/<html[^>]*>/i`
   handles both. If react-email adds `dir="ltr"` or other attributes to `<html>`, the greedy
   `[^>]*` capture handles it.

2. **MSO block placement:** The MSO block goes immediately after `<head>` — before any `<style>`
   or `<meta>` tags. Some guides place it after `<meta charset>`, but "immediately after `<head>`"
   is simpler and works correctly.

3. **Title escaping:** The newsletter title comes from user input (stored in DB). HTML entity
   escaping is applied to prevent `<`, `>`, `&`, `"` from breaking the `<title>` element.

4. **console.warn for missing patterns:** Rather than throwing, the function logs a warning and
   returns the unmodified HTML. This prevents a malformed react-email output from crashing the
   entire export. The exported HTML may not be fully Outlook-compatible in that edge case.

5. **Pure function:** No imports from React, no async. Takes string, returns string. Easy to
   unit test and reason about.
