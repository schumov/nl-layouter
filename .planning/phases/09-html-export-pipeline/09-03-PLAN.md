# Plan 09-03: documentToEmailTree — Full React-Email Component Tree

## Goal

Implement `documentToEmailTree()` — the function that assembles a complete react-email
component tree from a `NewsletterDoc`. Produces the tree passed to `render()` in Plan 09-04.
Handles all 5 layout types, MSO multi-column wrappers (EXPORT-04), and embeds header/footer
preset HTML (EXPORT-05). Turns the 3 RED `documentToEmailTree.test.ts` tests GREEN.

## Context

`documentToEmailTree` sits at the assembly layer of the pipeline. It receives:
- `doc: NewsletterDoc` — the full newsletter document from DB
- `headerHtml: string` — preset HTML fetched from DB (may be empty string)
- `footerHtml: string` — preset HTML fetched from DB (may be empty string)

It returns a `React.ReactElement` passed directly to `@react-email/render`'s `render()`.
The MSO conditional comment wrapping for multi-column layouts must be injected here because
react-email `Row`/`Column` components do not produce MSO VML wrappers automatically.

## Tasks

### Task 1: Create `documentToEmailTree.tsx`

Create `apps/server/src/export/documentToEmailTree.tsx`:

```tsx
// apps/server/src/export/documentToEmailTree.tsx
// Assembles the full react-email React tree from a NewsletterDoc.
//
// Architecture:
//   documentToEmailTree(doc, headerHtml, footerHtml)
//     └── <EmailDocument doc footerHtml headerHtml>
//           ├── <div dangerouslySetInnerHTML={headerHtml} />   (EXPORT-05)
//           ├── {doc.rows.map(row => <EmailRow row globalStyles />)}
//           └── <div dangerouslySetInnerHTML={footerHtml} />   (EXPORT-05)
//
// EmailRow handles per-row rendering:
//   1col  → <table><tr><td> wrapper (no MSO needed)
//   2col+ → MSO VML table wrapper + react-email layout columns (EXPORT-04)
//
// elementToEmail dispatches to the 5 element renderers.
import React from 'react';
import { EmailDocument } from './EmailDocument.js';
import {
  imageToEmail,
  imageLinkToEmail,
  buttonToEmail,
  richTextToEmail,
  dividerToEmail,
} from './elementRenderers.js';

// ── Type mirrors (from apps/client/src/types/newsletter.ts) ──────────────────

type LayoutType =
  | '1col'
  | '2col'
  | '3col'
  | 'small-left-big-right'
  | 'big-left-small-right';

interface ImageElement   { type: 'image';      id: string; src: string; alt: string; width?: string; }
interface ImageLinkElement { type: 'image-link'; id: string; src: string; alt: string; href: string; width?: string; }
interface ButtonElement  { type: 'button';     id: string; label: string; href: string; backgroundColor: string; textColor: string; borderRadius?: string; style: 'solid'|'outline'|'ghost'; }
interface RichTextElement { type: 'rich-text'; id: string; content: { type: 'doc'; content: unknown[] }; textStyle: 'header'|'subheader'|'body'|'code'; }
interface DividerElement { type: 'divider';   id: string; color: string; spacing: number; thickness: number; }

type ElementUnion = ImageElement | ImageLinkElement | ButtonElement | RichTextElement | DividerElement;

interface ColumnSlot { id: string; element: ElementUnion | null; }

interface Section {
  id: string;
  layoutType: LayoutType;
  slots: ColumnSlot[];
  backgroundColor?: string;
  paddingTop?: number;
  paddingBottom?: number;
}

interface GlobalStyles {
  fontFamily: string;
  backgroundColor: string;
  contentWidth: number;
  primaryColor: string;
}

interface NewsletterDoc {
  header: { presetId: string; variables: Record<string, string> };
  footer: { presetId: string; variables: Record<string, string> };
  rows: Section[];
  globalStyles: GlobalStyles;
  preHeader?: string;
}

// ── Column width map ─────────────────────────────────────────────────────────
// These are integer pixel values for the 600px content width.
// small-left-big-right: 33%/67% of 600px → 198px/396px (integer, sums to 594 not 600)
// Use 200px/400px for cleaner math, OR keep 198/396 per spec. Spec says 198/396.
const COLUMN_WIDTHS: Record<LayoutType, number[]> = {
  '1col':                   [600],
  '2col':                   [300, 300],
  '3col':                   [200, 200, 200],
  'small-left-big-right':   [198, 396],
  'big-left-small-right':   [396, 198],
};

// ── elementToEmail dispatcher ─────────────────────────────────────────────────

function elementToEmail(el: ElementUnion | null): React.ReactElement | null {
  if (!el) return null;
  switch (el.type) {
    case 'image':      return imageToEmail(el);
    case 'image-link': return imageLinkToEmail(el);
    case 'button':     return buttonToEmail(el);
    case 'rich-text':  return richTextToEmail(el);
    case 'divider':    return dividerToEmail(el);
    default:
      // assertNever pattern — exhaustive check; TypeScript will catch missing cases
      return null;
  }
}

// ── EmailRow component ────────────────────────────────────────────────────────

interface EmailRowProps {
  row: Section;
  globalStyles: GlobalStyles;
}

function EmailRow({ row, globalStyles }: EmailRowProps) {
  const widths = COLUMN_WIDTHS[row.layoutType] ?? [600];
  const isMultiCol = widths.length > 1;
  const contentWidth = globalStyles.contentWidth ?? 600;

  const rowStyle: React.CSSProperties = {
    backgroundColor: row.backgroundColor ?? 'transparent',
    paddingTop:    row.paddingTop    != null ? `${row.paddingTop}px`    : undefined,
    paddingBottom: row.paddingBottom != null ? `${row.paddingBottom}px` : undefined,
  };

  if (!isMultiCol) {
    // ── Single column (1col) — simple table/td, no MSO needed ────────────────
    const slot = row.slots[0];
    const el = slot ? elementToEmail(slot.element) : null;
    return (
      <table
        role="presentation"
        width={contentWidth}
        style={{ ...rowStyle, borderCollapse: 'collapse', width: `${contentWidth}px` }}
      >
        <tbody>
          <tr>
            <td
              width={contentWidth}
              valign="top"
              style={{ padding: '0', verticalAlign: 'top' }}
            >
              {el}
            </td>
          </tr>
        </tbody>
      </table>
    );
  }

  // ── Multi-column layout ───────────────────────────────────────────────────
  // Pattern: MSO VML table wraps for Outlook (EXPORT-04), with react-email
  // compatible structure for modern clients.
  //
  // MSO wrapper structure:
  //   <!--[if mso]><table role="presentation" width="600"><tr><td width="N" valign="top"><!--<![endif]-->
  //   {column content}
  //   <!--[if mso]></td><td width="M" valign="top"><!--<![endif]-->
  //   {column content}
  //   <!--[if mso]></tr></table><![endif]-->
  //
  // NOTE: dangerouslySetInnerHTML cannot be used for conditional comments because
  // React strips HTML comments. Instead, we use a workaround: inject the MSO comments
  // as raw HTML via a dangerouslySetInnerHTML span ONLY for the MSO markup, surrounding
  // the actual content with standard React elements.
  //
  // The safest approach for react-email + Outlook compatibility:
  // 1. Render column content via standard React (modern clients)
  // 2. Use dangerouslySetInnerHTML to inject <!--[if mso]--> comments as raw HTML strings
  //    via sibling <div> elements (juice will preserve HTML comments)

  const slots = row.slots.slice(0, widths.length);

  // Build the MSO conditional comment pieces as raw HTML strings.
  // These are injected via dangerouslySetInnerHTML into the rendered tree.
  const msoTableOpen = `<!--[if mso]><table role="presentation" width="${contentWidth}" style="border-collapse:collapse;"><tr><!--<![endif]-->`;
  const msoTableClose = `<!--[if mso]></tr></table><![endif]-->`;

  const msoColOpen = (width: number) =>
    `<!--[if mso]><td width="${width}" valign="top" style="padding:0;"><!--<![endif]-->`;
  const msoColClose = `<!--[if mso]></td><!--<![endif]-->`;

  return (
    <table
      role="presentation"
      width={contentWidth}
      style={{
        ...rowStyle,
        borderCollapse: 'collapse',
        width: `${contentWidth}px`,
      }}
    >
      <tbody>
        <tr>
          <td style={{ padding: '0' }}>
            {/* MSO table open */}
            <div dangerouslySetInnerHTML={{ __html: msoTableOpen }} />

            {/* Column layout — uses display:inline-block for modern clients.
                Each column is wrapped in its own MSO <td> via conditional comments. */}
            {slots.map((slot, idx) => {
              const width = widths[idx] ?? widths[0] ?? 300;
              const el = elementToEmail(slot?.element ?? null);
              return (
                <React.Fragment key={slot?.id ?? idx}>
                  {/* MSO column open */}
                  <div dangerouslySetInnerHTML={{ __html: msoColOpen(width) }} />

                  {/* Column content — display:inline-block for non-Outlook clients */}
                  <div
                    style={{
                      display: 'inline-block',
                      width: `${width}px`,
                      maxWidth: `${width}px`,
                      verticalAlign: 'top',
                    }}
                  >
                    <table
                      role="presentation"
                      width={width}
                      style={{ borderCollapse: 'collapse', width: `${width}px` }}
                    >
                      <tbody>
                        <tr>
                          <td valign="top" style={{ padding: '0', verticalAlign: 'top' }}>
                            {el}
                          </td>
                        </tr>
                      </tbody>
                    </table>
                  </div>

                  {/* MSO column close */}
                  <div dangerouslySetInnerHTML={{ __html: msoColClose }} />
                </React.Fragment>
              );
            })}

            {/* MSO table close */}
            <div dangerouslySetInnerHTML={{ __html: msoTableClose }} />
          </td>
        </tr>
      </tbody>
    </table>
  );
}

// ── Public API ───────────────────────────────────────────────────────────────

/**
 * Assemble the full react-email React tree from a newsletter document.
 *
 * @param doc        The newsletter document loaded from DB.
 * @param headerHtml Raw HTML string for the header preset (may be empty string).
 * @param footerHtml Raw HTML string for the footer preset (may be empty string).
 * @returns          A React.ReactElement ready for @react-email/render's render().
 */
export function documentToEmailTree(
  doc: NewsletterDoc,
  headerHtml: string,
  footerHtml: string,
): React.ReactElement {
  return (
    <EmailDocument doc={doc} headerHtml={headerHtml} footerHtml={footerHtml}>
      {/* Header preset HTML (EXPORT-05) — injected as raw HTML */}
      {headerHtml && (
        <div dangerouslySetInnerHTML={{ __html: headerHtml }} />
      )}

      {/* Content rows */}
      {(doc.rows ?? []).map((row) => (
        <EmailRow key={row.id} row={row} globalStyles={doc.globalStyles} />
      ))}

      {/* Footer preset HTML (EXPORT-05) — injected as raw HTML */}
      {footerHtml && (
        <div dangerouslySetInnerHTML={{ __html: footerHtml }} />
      )}
    </EmailDocument>
  );
}
```

**MSO Conditional Comments — Implementation Note:**

React strips HTML comments (`<!-- ... -->`) from JSX trees. The workaround used above is to
inject conditional comments via `dangerouslySetInnerHTML` on thin wrapper `<div>` elements.
This is the standard approach in transactional email templates using React.

After `render()` and `juice()`, the output HTML will contain the MSO comments as raw strings
because:
1. `dangerouslySetInnerHTML` output is injected verbatim into the React-rendered HTML string
2. `juice` (CSS inliner) preserves HTML comments
3. `wrapWithDoctype()` (Plan 09-05) does not modify comment nodes

**Alternative if dangerouslySetInnerHTML comments are stripped by react-email's renderer:**
Use string substitution in `pipeline.ts` — insert MSO comments after `render()` by replacing
marker strings (e.g., `data-mso-open="..."`) with the actual `<!--[if mso]>` comments.
Only use this fallback if the dangerouslySetInnerHTML approach fails in testing.

**Column width rationale:**
- `small-left-big-right`: 198px + 396px = 594px (not 600). The 6px gap allows for email client
  rendering margins. This matches the spec: "198/396 (33%/67% of 600 — use integer px)".
- `big-left-small-right`: 396px + 198px = same.
