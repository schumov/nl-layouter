# Plan 09-07: Export Button UI — BuilderHeader Wiring + Client Download

## Goal

Wire the Export button in `BuilderHeader.tsx` to the real export endpoint, replacing the
current `toast('Export is not yet available')` placeholder. Implements fetch + Blob download,
loading state, and error handling. Turns the 2 RED `BuilderHeader.export.test.tsx` tests
GREEN, completing Phase 9.

## Context

**Current state** (`BuilderHeader.tsx`, line ~158):
```tsx
<Button
  variant="outline"
  size="sm"
  onClick={() => toast('Export is not yet available')}
>
  Export
</Button>
```

**Target state:** The button calls `handleExport()` which:
1. Sets `isExporting = true` (disables button, shows "Exporting…")
2. Fetches `POST http://localhost:3001/newsletters/${id}/export`
3. On success: downloads response as a `.html` file, shows success toast
4. On failure: shows error toast
5. Sets `isExporting = false` in `finally`

**API URL:** The client uses direct `http://localhost:3001/...` calls (no Vite proxy, no
`/api` prefix). Confirmed by examining `apps/client/src/hooks/useNewsletters.ts`,
`useAutoSave.ts`, `usePresets.ts` — all use `http://localhost:3001` directly.

**Change 2 — Add `handleExport` function.** Insert after the `handleKeyDown` function
(and before the `return` statement):

```typescript
const handleExport = async () => {
  setIsExporting(true);
  try {
    const res = await fetch(`http://localhost:3001/newsletters/${id}/export`, {
      method: 'POST',
    });
    if (!res.ok) throw new Error(`Export failed: ${res.status}`);

    const html = await res.text();
    const blob = new Blob([html], { type: 'text/html' });
    const url  = URL.createObjectURL(blob);
    const a    = document.createElement('a');
    a.href     = url;
    // Slugify title for filename (mirrors server-side logic)
    a.download = `${title.replace(/[^a-z0-9]/gi, '-').replace(/-{2,}/g, '-').toLowerCase() || 'newsletter'}.html`;
    document.body.appendChild(a);
    a.click();
    document.body.removeChild(a);
    URL.revokeObjectURL(url);
    toast.success('Newsletter exported!');
  } catch {
    toast.error('Export failed. Please try again.');
  } finally {
    setIsExporting(false);
  }
};
```

**Important:** `document.body.appendChild(a)` + `a.click()` + `document.body.removeChild(a)`
is the robust cross-browser pattern. Some browsers require the `<a>` to be in the DOM to
trigger the download. Do NOT skip the `appendChild`/`removeChild` steps.

**Change 3 — Update the Export button JSX.** Find the current Export button:

The `\u2026` is the ellipsis character (…) — consistent with the "Saving…" text already used
