# Plan 09-05: render() + Juice Pipeline Orchestrator

## Goal

Implement `pipeline.ts` — the orchestrating function `renderToEmailHtml()` that chains the
four pipeline stages: build tree → render to HTML → inline CSS with juice → apply DOCTYPE/MSO
wrapper. Completing this plan turns all 16 server-side RED tests GREEN.

## Context

`renderToEmailHtml` is the single entry point called by the export API route (Plan 09-06).
It takes a `NewsletterDoc` plus pre-fetched header/footer HTML strings and returns a fully
self-contained `.html` string ready for download. This plan is the final server-side
pipeline step before the route and client wiring.

**Dependency chain:**
```
documentToEmailTree (09-03)
       ↓
   render() from @react-email/render
       ↓
   juice() CSS inliner
       ↓
wrapWithDoctype() from doctype.ts (09-04)
       ↓
   renderToEmailHtml → Promise<string>
```

## Tasks

### Task 1: Create `pipeline.ts`

Create `apps/server/src/export/pipeline.ts`:

```typescript
// apps/server/src/export/pipeline.ts
// Pipeline orchestrator: converts a NewsletterDoc to a self-contained HTML email string.
//
// Four-stage pipeline:
//   1. documentToEmailTree   — assemble react-email React tree
//   2. render()              — React → raw HTML string (via @react-email/render)
//   3. juice()               — inline CSS, remove <style> blocks
//   4. wrapWithDoctype()     — XHTML DOCTYPE + VML namespaces + MSO head block
//
// This is the ONLY function called by the export API route (routes/export.ts).
import { render } from '@react-email/render';
import juice from 'juice';
import { documentToEmailTree } from './documentToEmailTree.js';
import { wrapWithDoctype } from './doctype.js';

// ── Type mirror ───────────────────────────────────────────────────────────────

type LayoutType =
  | '1col' | '2col' | '3col' | 'small-left-big-right' | 'big-left-small-right';

interface Section {
  id: string;
  layoutType: LayoutType;
  slots: Array<{ id: string; element: unknown }>;
  backgroundColor?: string;
  paddingTop?: number;
  paddingBottom?: number;
}

interface NewsletterDoc {
  header: { presetId: string; variables: Record<string, string> };
  footer: { presetId: string; variables: Record<string, string> };
  rows: Section[];
  globalStyles: {
    fontFamily: string;
    backgroundColor: string;
    contentWidth: number;
    primaryColor: string;
  };
  preHeader?: string;
}

// ── Pipeline ─────────────────────────────────────────────────────────────────

/**
 * Convert a NewsletterDoc to a self-contained, email-client-ready HTML string.
 *
 * @param doc        The newsletter document (loaded from DB via routes/export.ts).
 * @param headerHtml Preset HTML for the email header. Pass empty string if not available.
 * @param footerHtml Preset HTML for the email footer. Pass empty string if not available.
 * @param title      Newsletter title — used for the HTML <title> element and filename.
 * @returns          Promise<string> — fully self-contained HTML email.
 */
export async function renderToEmailHtml(
  doc: NewsletterDoc,
  headerHtml: string,
  footerHtml: string,
  title = 'Newsletter',
): Promise<string> {
  // ── Stage 1: Build React tree ─────────────────────────────────────────────
  // documentToEmailTree assembles EmailDocument + header div + rows + footer div.
  // Pre-header span injection happens inside EmailDocument (Plan 09-01).
  // MSO conditional comments for multi-column rows are in the row elements (Plan 09-03).
  const tree = documentToEmailTree(doc as never, headerHtml, footerHtml);

  // ── Stage 2: Render React tree → HTML string ──────────────────────────────
  // @react-email/render's render() returns a Promise<string> in v2.x.
  // The raw output contains:
  //   - <!DOCTYPE html> (HTML5)
  //   - <html lang="en">
  //   - Inline styles from React's style prop
  //   - Possibly a <style> block in <head> with reset/base CSS
  //   - dangerouslySetInnerHTML content (MSO comments, header/footer HTML)
  const rawHtml = await render(tree, { pretty: false });

  // ── Stage 3: Inline CSS with juice ────────────────────────────────────────
  // juice reads any <style> blocks from the HTML, inlines the rules into style=""
  // attributes on matching elements, then removes the <style> blocks.
  //
  // Options:
  //   removeStyleTags: true      — removes <style> blocks after inlining (EXPORT-03)
  //   preserveMediaQueries: false — strip @media queries (not needed for static email)
  //   xmlMode: false             — HTML mode (not XHTML — we add DOCTYPE in step 4)
  //   applyAttributesTableElements: false — don't convert style to presentational attrs
  //   preserveFontFaces: false   — strip @font-face (not email-safe)
  //   preserveKeyFrames: false   — strip @keyframes (not email-safe)
  const inlinedHtml = juice(rawHtml, {
    removeStyleTags: true,
    preserveMediaQueries: false,
    xmlMode: false,
    applyAttributesTableElements: false,
    preserveFontFaces: false,
    preserveKeyFrames: false,
  });

  // ── Stage 4: Replace DOCTYPE + inject MSO head block ─────────────────────
  // wrapWithDoctype replaces HTML5 DOCTYPE with XHTML Transitional,
  // adds VML namespace attrs to <html>, sets <title>, and injects MSO reset block.
  const finalHtml = wrapWithDoctype(inlinedHtml, title);

  return finalHtml;
}
```

**Implementation notes:**

1. **`render()` API in @react-email/render@^2.x:**
   The v2 API returns `Promise<string>`. If the installed version is v1.x (returns string
   synchronously), remove `await` and `async` from the function. Check with:
   ```bash
   cd apps/server && node --input-type=module -e "import { render } from '@react-email/render'; console.log(typeof render);"
   ```
   In v1.x, `render` returns `string` directly. In v2.x+, it returns `Promise<string>`.
   The function signature above uses `await` — if v1.x is installed, adjust accordingly.

2. **juice import:** `juice` is a CommonJS package. In ESM context, use default import:
   `import juice from 'juice'`. If this fails with "does not provide an export named 'default'",
   use: `import { createRequire } from 'module'; const require = createRequire(import.meta.url); const juice = require('juice');`

3. **`removeStyleTags: true`:** This is the critical setting for EXPORT-03. react-email may
   generate a `<style>` block in `<head>` with CSS reset rules. juice inlines those rules into
   `style=""` attrs and removes the block. Without this, Gmail strips the `<style>` block and
   loses the styles.

4. **`preserveMediaQueries: false`:** Strips `@media` queries since we're producing a static
   fixed-width email. If responsive email is needed in a future phase, re-enable this.

5. **`doc as never` cast:** The `NewsletterDoc` type here omits full `ElementUnion` details
   on `rows[].slots[].element`. The cast `as never` passes TypeScript without requiring
   the full deep type. `documentToEmailTree` handles the full type internally.
