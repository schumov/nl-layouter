# Plan 09-01: Package Setup + EmailDocument Component

## Goal

Install the server-side React + react-email + juice packages and create the top-level
`EmailDocument` component that forms the outer wrapper of every exported email. Provides
the HTML skeleton (DOCTYPE injection comes in Plan 09-05), pre-header span injection
(EXPORT-06), and global background/width styles.

## Context

The server has no React installed. This plan adds the required packages to `apps/server`
and creates the `src/export/` directory with the first file. `EmailDocument` is the root
component consumed by `documentToEmailTree` in Plan 09-03.

## Tasks

### Task 1: Install Server-Side Packages

Run the following from the monorepo root (pnpm workspace):

```bash
cd apps/server
pnpm add react@^19.0.0 react-dom@^19.0.0 @react-email/components@^1.0.12 @react-email/render@^2.0.8 juice@^12.1.0
pnpm add -D @types/react@^19.0.0 @types/react-dom@^19.0.0
```

After installation, verify `apps/server/package.json` has these in dependencies:
- `react`, `react-dom`, `@react-email/components`, `@react-email/render`, `juice`

And in devDependencies:
- `@types/react`, `@types/react-dom`

**Vitest compatibility check:** After installing, run:
```bash
cd apps/server && pnpm test 2>&1 | head -30
```
The server tests should still fail (RED — modules not found), but vitest itself must not crash. If vitest crashes due to peer deps, resolve peer dep warnings before proceeding.

**react-email version note:** If `@react-email/components@^1.0.12` or `@react-email/render@^2.0.8`

Check with: `pnpm info @react-email/components version` and `pnpm info @react-email/render version`.

### Task 2: Create EmailDocument Component

Create `apps/server/src/export/EmailDocument.tsx`:

```tsx
// apps/server/src/export/EmailDocument.tsx
// Top-level email document component for Phase 9 HTML export pipeline.
//
// Responsibilities:
//   - Provides the react-email Html/Head/Body skeleton
//   - Injects pre-header hidden span immediately after <Body> open (EXPORT-06)
//   - Applies globalStyles.backgroundColor to the Body background
//   - Centers content in a 600px (or contentWidth) wrapper table
//
// This component does NOT inject DOCTYPE or MSO head tags — that is handled
// by wrapWithDoctype() in Plan 09-05 via string manipulation after render().
import React from 'react';
import { Html, Head, Body, Section, Container } from '@react-email/components';

// ── Types (duplicated from apps/client/src/types/newsletter.ts — server is standalone) ──

interface GlobalStyles {
  fontFamily: string;
  backgroundColor: string;
  contentWidth: number;
  primaryColor: string;
}

interface NewsletterDoc {
  header: { presetId: string; variables: Record<string, string> };
  footer: { presetId: string; variables: Record<string, string> };
  rows: unknown[];
  globalStyles: GlobalStyles;
  preHeader?: string;
}

// ── Component ────────────────────────────────────────────────────────────────

export interface EmailDocumentProps {
  doc: NewsletterDoc;
  headerHtml: string;
  footerHtml: string;
  children: React.ReactNode;
}

export function EmailDocument({ doc, children }: EmailDocumentProps) {
  const { globalStyles } = doc;
  const contentWidth = globalStyles.contentWidth ?? 600;

  return (
    <Html lang="en">
      <Head>
        {/* Meta charset — react-email may add this; kept explicit for safety */}
        <meta httpEquiv="Content-Type" content="text/html; charset=UTF-8" />
        <meta name="viewport" content="width=device-width, initial-scale=1.0" />
        <title></title>
      </Head>
      <Body
        style={{
          backgroundColor: globalStyles.backgroundColor,
          fontFamily: globalStyles.fontFamily,
          margin: '0',
          padding: '0',
        }}
      >
        {/* Pre-header: hidden span shown in inbox preview only (EXPORT-06).
            Injected only when preHeader text is present.
            Styled with display:none so it does not appear in the rendered email body.
            Uses opacity:0 + maxHeight:0 + overflow:hidden as belt-and-suspenders.
        */}
        {doc.preHeader && doc.preHeader.trim().length > 0 && (
          <span
            style={{
              display: 'none',
              fontSize: '1px',
              color: '#ffffff',
              maxHeight: '0',
              overflow: 'hidden',
              opacity: 0,
              // Zero-width non-joiner padding prevents preview bleed in some clients
            }}
            aria-hidden="true"
          >
            {doc.preHeader}
            {/* Pad to ~90 chars with zero-width non-joiners to prevent Gmail preview bleed */}
            {'\u200C\u200B\u200C\u200B\u200C\u200B\u200C\u200B\u200C\u200B'}
          </span>
        )}

        {/* Center wrapper: 600px (contentWidth) centered table */}
        <table
          role="presentation"
          width="100%"
          style={{ borderCollapse: 'collapse', margin: '0', padding: '0' }}
        >
          <tbody>
            <tr>
              <td align="center" style={{ padding: '0' }}>
                {/* Content container — constrains to contentWidth */}
                <table
                  role="presentation"
                  width={contentWidth}
                  style={{
                    borderCollapse: 'collapse',
                    width: `${contentWidth}px`,
                    maxWidth: `${contentWidth}px`,
                  }}
                >
                  <tbody>
                    <tr>
                      <td style={{ padding: '0' }}>
                        {children}
                      </td>
                    </tr>
                  </tbody>
                </table>
              </td>
            </tr>
          </tbody>
        </table>
      </Body>
    </Html>
  );
}

export default EmailDocument;
```

**Implementation notes:**

1. **`@react-email/components` API:** Import `Html`, `Head`, `Body` from `@react-email/components`.
   If `Container` is not available in the installed version, replace with `<table>` (already done above).
   Do NOT use `Section` or `Row` from react-email for the outer wrapper — those produce table rows,
   not the centering wrapper. Use raw `<table>` elements for the outer centering structure.

2. **Pre-header styling:** All 5 CSS properties are required:
   - `display: 'none'` — hides from most clients
   - `fontSize: '1px'` — reduces visual footprint
   - `color: '#ffffff'` — matches typical white background
   - `maxHeight: '0'` — belt-and-suspenders for preview height
   - `overflow: 'hidden'` — prevents layout impact
   - `opacity: 0` — extra hiding for some clients

3. **contentWidth default:** If `doc.globalStyles.contentWidth` is falsy, use `600` as default.

4. **No logo/header/footer rendering here:** `EmailDocument` only provides the outer skeleton.
   The `children` prop receives the header HTML div + row tree + footer HTML div from
   `documentToEmailTree` in Plan 09-03.

5. **Type duplication is intentional:** The server does not import from `apps/client/src/types/`.
   The local `NewsletterDoc` interface must match the shape — keep it in sync manually.
   The `rows: unknown[]` is fine here since EmailDocument doesn't iterate rows directly.
