**Edit 2 — Add `updateElement` implementation** immediately after the `removeElement` implementation (before the `setElement` JSDoc block):

Insert this block:
```typescript
updateElement: (slotId, patch) =>
  set((state) => {
    if (!state.doc) return;
    for (const row of state.doc.rows) {
      for (const slot of row.slots) {
        if (slot.id === slotId && slot.element) {
          Object.assign(slot.element, patch);  // Immer in-place mutation — do NOT use spread
          return;  // early exit on first match
        }
      }
    }
    // slotId not found or slot.element is null — silent no-op
  }),
```

Note the double nested loop: `addElement` uses `row.slots.find()` (single level), but `updateElement` needs `for...of row.slots` to allow early `return` from the outer loop via the inner guard. Either nested `find` or nested `for...of` pattern is acceptable as long as the early return exits cleanly.
