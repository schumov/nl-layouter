## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | shadcn/ui | `apps/client/components.json` вҖ” confirmed |
| Style | new-york | `components.json` style: "new-york" |
| Preset | neutral base, CSS variables, no prefix | `components.json` baseColor: neutral |
| Component library | Radix UI (via shadcn) | Phase 1 foundation вҖ” carry-forward |
| Icon library | lucide-react | `components.json` iconLibrary: lucide |
| Font | System default (sans-serif stack) | `index.css` вҖ” no fontFamily token override |
| CSS engine | Tailwind CSS v4 вҖ” CSS-first, `@theme inline` tokens, no `tailwind.config.*` | Carry-forward from Phase 1 |

**Tailwind v4 JIT rule (locked вҖ” carry-forward):** All class names MUST be complete string literals. Never construct via template literals or dynamic concatenation. Applies to every new class in Phase 6.

**font-medium FORBIDDEN (locked вҖ” carry-forward):** `font-medium` (weight 500) is not used anywhere in the project. Only `font-semibold` (600) for emphasis. Violations cause Tailwind v4 purge issues in this project.

**Inline styles for configurable colours (CC-2/CC-6):** `backgroundColor`, `textColor`, `borderRadius`, and any other user-configured property that must survive email export MUST be set via `style={}` prop вҖ” NOT Tailwind classes. This applies to `ButtonRenderer` and any canvas element that carries email-destined styles.

**New shadcn components required:** None. `Input` and `Button` are already installed from Phases 2вҖ“5. Color picking uses native HTML5 `<input type="color">` вҖ” no registry install needed.

## Spacing Scale

Carries forward Phase 5 scale. Phase 6 adds editor-specific usages:

| Token | px | Tailwind | Phase 6 Usage |
|-------|----|----------|---------------|
| xs | 4px | `gap-1`, `p-1`, `top-1`, `right-1`, `p-0.5` | LabelвҶ”input gap (`gap-1`); ExternalLink badge position (`top-1 right-1`); badge inner padding (`p-0.5`) |
| sm | 8px | `gap-2`, `py-2` | Color picker: swatchвҶ”hex input gap (`gap-2`); style toggle gap between buttons (`gap-1` exception вҖ” see below) |
| md | 16px | `p-4`, `px-4` | Editor body padding (`p-4`); ImageRenderer padding on empty-state (`p-4`); InspectorPanel header horizontal padding (`px-4`) вҖ” carry-forward Phase 5 |
| lg | 24px | `py-6` | Not used in Phase 6 components |
| xl | 32px | `py-8` | Canvas vertical padding вҖ” carry-forward Phase 3 |
| 2xl | 48px | `h-48` | Empty canvas drop zone height вҖ” carry-forward Phase 4 |
| 3xl | 64px | `py-16` | Empty canvas loading state padding вҖ” carry-forward Phase 3 |

**Phase 6 exceptions:**

| Element | Value | Rationale |
|---------|-------|-----------|
| ColumnSlot minimum height | 80px вҖ” `min-h-[80px]` | Phase 3 baseline вҖ” unchanged |
| Image slot minimum height | 70px вҖ” `min-h-[70px]` | D-02: prevents zero-height collapse; shorter than ColumnSlot since images render their own visual content |
| Inspector header padding | `py-3` (12px) | Phase 5 carry-forward exception вҖ” matches shadcn panel conventions |
| Editor field groups gap | `gap-3` (12px) | 12px between field groups in editor body: tighter than `md` (16px) suits compact inspector, wider than `sm` (8px) provides clear field separation |
| Section divider margin | `my-3` (12px) | Same 12px rhythm as field groups gap |
| Style toggle button gap | `gap-1` (4px) | Segmented control: buttons sit flush with minimal gap, not the standard 8px element gap |
| Color swatch size | 24Г—24px вҖ” `w-6 h-6` | On the 4px grid (6 units); standard native `<input type="color">` minimum usable click target |
| Hex input width | 88px вҖ” `w-[88px]` | Fits 7-char hex + cursor comfortably in monospace; not a token вҖ” explicit pixel value |
| Button padding (renderer) | `12px 24px` вҖ” inline style | ButtonRenderer inline style; top/bottom 12px, left/right 24px; 12 is on the 4px grid |

## Typography

Carries forward Phase 5 scale. No new type roles. Phase 6 introduces editor field labels at `text-xs`:

| Role | Size | Tailwind | Weight | Line Height | Phase 6 Usage |
|------|------|----------|--------|-------------|---------------|
| Body | 14px | `text-sm` | 400 regular | 1.5 | Editor Input field values; inspector note text; image empty-state body text |
| Label / UI title | 14px | `text-sm` | 600 semibold | 1.2 | InspectorPanel header element name; button label in renderer |
| Field label | 12px | `text-xs` | 600 semibold | 1.2 | Editor field labels ("Source URL", "Alt Text", "Width", "Label", "Link URL", "Background Color", "Text Color", "Style") |
| Hint / muted | 12px | `text-xs` | 400 regular | 1.5 | Image empty-state "Add image URL" text; color hex hint |

**Two weights only:** `400` for body/hint/muted text; `600 semibold` for all labels, field group headers, and interactive labels. Weight 500 is never used.

**Hex input typography:** `className="font-mono text-xs"` вҖ” monospace for hex legibility, 12px matches field label size.

### 1. `ImageRenderer`

**File:** `apps/client/src/components/builder/ImageRenderer.tsx`

**Purpose:** Canvas display for `ImageElement`. Shows branded placeholder when `src` is empty; renders `<img>` when `src` is non-empty.

**Props:**
```typescript
interface ImageRendererProps {
  element: ImageElement;
}
```

**Empty-state (src === ''):**
```
div.min-h-[70px].flex.flex-col.items-center.justify-center.bg-accent.rounded.gap-1.p-4
  в”ңв”Җв”Җ <Image className="size-6 text-muted-foreground" />        // lucide Image icon
  в””в”Җв”Җ <span className="text-xs text-muted-foreground">Add image URL</span>
```

Full classes:
```typescript
'min-h-[70px] flex flex-col items-center justify-center bg-accent rounded gap-1 p-4'
```

**Rendered image (src !== ''):**
```typescript
<img
  src={element.src}
  alt={element.alt}
  className="block min-h-[70px] w-full"
  style={{ width: element.width ?? '100%', objectFit: 'cover', display: 'block' }}
/>
```

Note: `width` is set via inline style (CC-6: export compatibility). `className="block"` prevents inline baseline gap. `min-h-[70px]` prevents collapsed slot before image loads.

**State matrix:**

| State | Condition | Visual |
|-------|-----------|--------|
| Empty placeholder | `element.src === ''` | `bg-accent` + Image icon + "Add image URL" text |
| Rendered | `element.src !== ''` | `<img>` with objectFit:cover, width from element or 100% default |

### 3. `ButtonRenderer`

**File:** `apps/client/src/components/builder/ButtonRenderer.tsx`

**Purpose:** Canvas display for `ButtonElement`. Full-width block button, two style variants (solid/outline), all configurable colours via inline styles (CC-2/CC-6, D-12).

**Props:**
```typescript
interface ButtonRendererProps {
  element: ButtonElement;
}
```

**Tag:** `<a>` (not `<button>`) вҖ” consistent with email export output. `href={element.href || '#'}` prevents navigation when href is empty.

**Outer wrapper:**
```typescript
<div className="w-full py-2 px-4">
```
Centres the button within the column slot with consistent vertical breathing room.

**Solid variant** (`element.style === 'solid'`):
```typescript
<a
  href={element.href || '#'}
  target="_blank"
  rel="noopener noreferrer"
  style={{
    display: 'block',
    width: '100%',
    textAlign: 'center',
    padding: '12px 24px',
    borderRadius: element.borderRadius ?? '4px',
    backgroundColor: element.backgroundColor,
    color: element.textColor,
    border: 'none',
    textDecoration: 'none',
    fontWeight: 600,
    fontSize: '14px',
    boxSizing: 'border-box',
  }}
>
  {element.label}
</a>
```

**Outline variant** (`element.style === 'outline'`):
```typescript
<a
  href={element.href || '#'}
  target="_blank"
  rel="noopener noreferrer"
  style={{
    display: 'block',
    width: '100%',
    textAlign: 'center',
    padding: '12px 24px',
    borderRadius: element.borderRadius ?? '4px',
    backgroundColor: 'transparent',
    color: element.backgroundColor,
    border: `2px solid ${element.backgroundColor}`,
    textDecoration: 'none',
    fontWeight: 600,
    fontSize: '14px',
    boxSizing: 'border-box',
  }}
>
  {element.label}
</a>
```

Note: Outline variant derives border colour and text colour from `element.backgroundColor` (D-12). There is no separate `borderColor` field on `ButtonElement`.

**Ghost variant:** `element.style === 'ghost'` exists in the type system but is NOT rendered in Phase 6 (D-05). If the switch encounters `ghost`, fall through to solid rendering as a safe default. Do NOT add ghost to the ButtonEditor toggle.

**State matrix:**

| `ghost` | _(falls through to solid вҖ” not in Phase 6 editor)_ | вҖ” | вҖ” |

### 9. `useNewsletterStore` вҖ” `updateElement` action

**File:** `apps/client/src/store/useNewsletterStore.ts`

**New action:**
```typescript
// In NewsletterActions interface:
updateElement: (slotId: string, patch: Partial<ElementUnion>) => void;

// In immer store implementation:
updateElement: (slotId, patch) =>
  set((state) => {
    if (!state.doc) return;
    for (const row of state.doc.rows) {
      for (const slot of row.slots) {
        if (slot.id === slotId && slot.element) {
          Object.assign(slot.element, patch);
          return;
        }
      }
    }
  }),
```

Note: `Object.assign(slot.element, patch)` is the correct Immer pattern for merging a patch into a nested object. Do NOT use `structuredClone` without `current()` вҖ” Immer proxies are not cloneable directly. Early `return` after first match avoids scanning all remaining slots.

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Image alt text | `<img alt={element.alt}>` вҖ” editor enforces `alt` field via `Alt Text` input |
| ImageLinkRenderer anchor | `<a href rel="noopener noreferrer" target="_blank">` вҖ” standard accessible link |
| ExternalLink badge | `aria-hidden="true"` on badge span вҖ” decorative; screen reader reads link href |
| ButtonRenderer anchor | `<a href target="_blank" rel="noopener noreferrer">` вҖ” semantically correct for email links |
| Editor field labels | All `<label>` elements use `htmlFor` or wrap the `<Input>` (planner chooses `htmlFor` pattern for explicit association) |
| Color picker native input | `aria-label="Background color swatch"` / `aria-label="Text color swatch"` on `<input type="color">` |
| Style toggle buttons | `aria-pressed={element.style === 'solid'}` on Filled; `aria-pressed={element.style === 'outline'}` on Outline |
| Back arrow | `aria-label="Back to palette"` вҖ” carry-forward Phase 5 |

## Registry Safety

| Registry | Components Used | Safety Gate |
|----------|----------------|-------------|
| shadcn official | `Button` (installed Phase 2), `Input` (installed Phase 2) | Not required вҖ” official registry, already installed |
| npm packages | `lucide-react` (Image, ExternalLink already in project) | npm packages вҖ” not shadcn registry; reviewed Phase 1/5 |
| Native HTML | `<input type="color">` | Browser native вҖ” no registry install; no vetting needed |

**No new shadcn components required.** All needed components (`Button`, `Input`) are already installed from Phases 2вҖ“5. Color picking uses native HTML5 `<input type="color">` вҖ” zero additional installs or vetting gates.

**No third-party shadcn registries.** No registry vetting gate required.

## Visual Focal Point

**Primary visual anchor (Phase 6 delivered state):** Real element renderers replace the `[image]` / `[button]` text stubs. Images with src fill their slots visually; buttons display with brand colours. The canvas progressively looks like a real newsletter preview.

**During canvas click (element selected):** The `ring-2 ring-ring ring-inset` selection ring on the slot + InspectorPanel opening with real editor fields (replacing the "Editing available in the next step." note) вҖ” the right panel transforms from placeholder to productive editor.

**ButtonRenderer as designed anchor:** A dropped button renders immediately with `#0066cc` background and "Click me" label вҖ” the most visually impactful default. Users understand the element is interactive and configurable at first glance.
