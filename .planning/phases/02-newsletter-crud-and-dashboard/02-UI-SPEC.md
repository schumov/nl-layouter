The shadcn `neutral` base color sets `--primary` to near-black (`oklch(0.205 0 0)`). Phase 2
must override this to **Infineon Blue** (#0066cc) so primary action buttons render in brand color.

**Edit `apps/client/src/index.css` — inside `:root {}`:**

```css

--primary: oklch(0.484 0.209 257.0);      /* #0066cc — Infineon Blue */
--primary-foreground: oklch(0.985 0 0);   /* white text on blue — unchanged */
```

## Spacing Scale

Standard 8-point grid. All values are multiples of 4.

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `gap-1`, `p-1` | Icon gaps, badge padding, inline tight spacing |
| sm | 8px | `gap-2`, `p-2` | Compact spacing between related items |
| md | 16px | `gap-4`, `p-4` | Default card padding, dialog padding, form gaps |
| lg | 24px | `gap-6`, `p-6` | Card grid gap, section padding within page |
| xl | 32px | `gap-8`, `p-8` | Page content vertical padding (`py-8`) |
| 2xl | 48px | `gap-12` | Major layout separation |
| 3xl | 64px | `gap-16` | Not used in Phase 2 |

**Exceptions:**
- `BuilderHeader` height: **56px** (`h-14`) — standard toolbar touch target height; not from 8-point scale
- Minimum touch target size: **44×44px** on all interactive icon buttons (⋮ menu trigger, ← back button)
- Page horizontal padding: **24px** (`px-6`) on content wrapper
- Dashboard max-width: **72rem** (`max-w-6xl`) — caps card grid at 1152px

## Typography

Exactly 4 sizes, 2 weights. No exceptions.

| Role | Size | Tailwind | Weight | Weight Value | Line Height | Usage |
|------|------|----------|--------|-------------|-------------|-------|
| Metadata | 12px | `text-xs` | Regular | 400 | 1.4 | Card timestamp, card section count |
| Body | 14px | `text-sm` | Regular | 400 | 1.5 | Dialog body, labels, dropdown items, toast text, save status |
| Heading-sm | 16px | `text-base` | Semibold | 600 | 1.25 | Card title, BuilderHeader title, dialog title, alert-dialog title |
| Heading-lg | 20px | `text-xl` | Semibold | 600 | 1.2 | Dashboard page heading ("Newsletters") |

**Rules:**
- Body (14px/400) and Heading-sm (16px/600) are the two most-used sizes in Phase 2 components.
- Metadata (12px/400) is only used for supporting info — never for primary content.
- No italic, no underline in UI chrome (underline reserved for inline links only).
- `font-mono` is **not** used in Phase 2.

## Interaction Patterns

### 1. Card Hover → ⋮ Menu Reveal

```
STATE: default
  Card: shadow-sm, border
  ⋮ button: opacity-0, pointer-events-none

ON HOVER (group-hover):
  ⋮ button: opacity-100, pointer-events-auto
  transition: opacity 150ms ease

ON ⋮ CLICK (stopPropagation):
  DropdownMenu opens
  Items: "Rename", separator, "Delete" (text-destructive)

ON CARD CLICK (outside ⋮ button):
  navigate('/newsletters/:id')
```

Tailwind group pattern:
```tsx
<div className="relative group">
  {/* card content */}
  <Button
    variant="ghost"
    size="icon"
    className="absolute top-2 right-2 h-8 w-8 opacity-0 group-hover:opacity-100 transition-opacity"
  >
    <MoreVertical className="h-4 w-4" />
  </Button>
</div>
```

### 2. Create Dialog Flow

```
CLOSED → OPEN (click "New Newsletter")
  Dialog renders, input autofocused
  Input value: "" (empty)
  Create button: disabled

ON INPUT CHANGE:
  Input empty (after trim): Create button disabled
  Input non-empty: Create button enabled

ON "Create" CLICK (or Enter key in input):
  Mutation fires (POST /newsletters)
  Create button: disabled + loading state
  Input: disabled

  ON SUCCESS:
    navigate('/newsletters/:id')  [dialog unmounts]

  ON ERROR:
    Create button: re-enabled
    Input: re-enabled
    No inline error shown (network errors are rare; silent retry by user)

ON "Cancel" or overlay click:
  Dialog closes, state resets
```

### 3. Delete Newsletter Flow

```
DROPDOWN: "Delete" clicked
  AlertDialog opens

ALERT DIALOG:
  Title: "Delete newsletter?"
  Body: `This will permanently delete "${title}". This action cannot be undone.`
  Buttons: [Keep Newsletter] [Delete Newsletter — destructive]

ON "Delete":
  AlertDialog closes immediately
  Optimistic: card removed from grid (no animation — instant)
  Toast appears: `"${title}" deleted` + [Undo] button (duration: 5000ms)
  DELETE API call scheduled for 5s delay

  IF "Undo" clicked before 5s:
    DELETE request cancelled (clearTimeout)
    Card restored to grid (from snapshot)
    Toast dismissed

  IF 5s elapsed without undo:
    DELETE API request fires

    ON DELETE SUCCESS:
      Nothing (already removed from grid)

    ON DELETE ERROR:
      Card restored to grid
      Error toast: `Failed to delete "${title}"`

ON "Cancel":
  AlertDialog closes, no action
```

### 4. BuilderHeader Inline Title Edit

```
DISPLAY MODE:
  <span> or <div> containing title text
  className: "text-base font-semibold cursor-pointer hover:underline"
  Max-width: max-w-sm or max-w-md — truncate with overflow-hidden

ON CLICK:
  Switch to EDIT MODE
  <input> replaces <span>
  className: "bg-transparent border-none shadow-none outline-none
              focus:ring-1 focus:ring-ring rounded text-base font-semibold
              min-w-[12rem] max-w-md"
  autoFocus + select all text on focus

EDIT MODE:
  User types new title

ON ENTER:
  If value.trim() === "" → revert to original title, switch to DISPLAY MODE
  If value.trim() === original → switch to DISPLAY MODE, no mutation
  If value.trim() !== original → fire PATCH mutation → DISPLAY MODE

ON BLUR:
  Same rules as ON ENTER

ON ESCAPE:
  Revert to original title immediately
  Switch to DISPLAY MODE
  No mutation fired

PATCH MUTATION:
  Invalidates both ['newsletters'] list and ['newsletter', id] query keys
  No toast — title update is inline visual confirmation
```

### 5. Save Status Indicator

```
STATES & TRANSITIONS:

  idle → (user makes first edit in session) → debouncing
  debouncing → (1500ms no further edits) → saving
  saving → (PUT succeeds) → saved
  saved → (3000ms) → idle   [D-11: fade after 3s]
  saving → (PUT fails) → error
  error → (5000ms auto-retry) → saving  [D-09]
  error → (user edits again) → debouncing

RENDERING:
  idle:      nothing rendered  [D-10]
  debouncing: nothing rendered (debounce is silent)
  saving:    <span className="text-sm text-muted-foreground">Saving…</span>
  saved:     <span className="text-sm text-muted-foreground animate-fade-out">Saved ✓</span>
  error:     <span className="text-sm text-destructive">Save failed</span>
```

> **animate-fade-out implementation:** Use CSS transition on opacity for the saved → idle transition. Simplest approach: `setTimeout` to set `saveStatus` back to `'idle'` after 3000ms (already in RESEARCH.md Pattern 7). The transition effect can be `transition-opacity duration-700` with opacity toggled.
